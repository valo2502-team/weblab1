# 0001 - Вибір архітектурного стилю

**Статус:** Прийнято

## Контекст
Проєкт - невеликий онлайн-магазин. Один розробник, обмежений час та обсяг. Потрібно швидко створити робочу систему та забезпечити зрозумілу структуру коду.

## Рішення
Вибираємо **модульний моноліт**:
- Шарова структура: api/ -> service/ -> domain/
- Єдина кодова база та одна база даних

## Розглянуті альтернативи
1. **Мікросервіси** - окремі сервіси для store, orders, users
   - Плюси: незалежне розгортання
   - Мінуси: складна підтримка для малого проєкту
2. **Звичайний моноліт** - все в одному шарі
   - Плюси: просто розгортати
   - Мінуси: важко масштабувати та підтримувати

## Наслідки
- Легке тестування та розвиток для одного розробника
- Чіткий розподіл відповідальності
