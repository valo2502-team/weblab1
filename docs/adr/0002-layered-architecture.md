# 0002 - Шарова архітектура

**Статус:** Прийнято

## Шари
1. **api/** - приймає HTTP-запити, перетворює в DTO, повертає відповіді
2. **service/** - бізнес-логіка, use-case'и, валідація, мапінг DTO <-> domain
3. **domain/** - сутності та інваріанти (Item, Order, OrderItem, Category)

## Залежності
- api -> service -> domain
- service може звертатися до domain
- domain незалежний від інших шарів

## Заборонені зв’язки
- api не звертається напряму в БД
- service не викликає api
- domain не залежить від service або api
